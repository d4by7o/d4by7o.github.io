应用宝APP爬虫

  • python
  • scrapy
  • 爬虫
Android

最近由于科研需要，需要爬一些国内的应用市场，收集一批app，github上有人做过这个事情，android-apps-crawler
这是一个基于scrapy的爬虫，项目完成了基本的功能，获得url放到数据库，多线程下载。但是对于科研需求来说，获得的信息当然越多越好<(￣ˇ￣)/，仅仅一个url怎么能满足呢。

很久之前写过一些爬虫，但是没有接触过scrapy，大概研究了下，感觉这是个黑科技，从此妈妈再也不用担心我不会写爬虫了，寻找链接的逻辑都已经内置，只需要修改过滤逻辑就行。

应用宝的应用详细信息页面里提供了很丰富的信息，包括应用名，下载量，类别，文件大小等等，这些信息都可能用得到。我们要做的事情就是把这些东西都爬下来。

这些文件大概的分工是这样的：
items.py 用来定义一个Item的数据结构，声明Item里有哪些内容，可以在这里定义你要爬的信息变量。
pipelines.py 用来处理sipder获取的Item,我们可以在这里把爬到的内容写进数据库。
settings.py 用来定义一些配置信息
spider文件夹里放的是主要的爬虫逻辑的代码，讨论的是爬哪里，爬什么
此外，还可以添加一个middlewares.py,这个文件的主要作用是Hook了url抓取的过程，你可以再里面过滤你不想爬的url。

2.2 selector

spider通过 XPath 或者 CSS 表达式来选择你想要的信息，推荐使用XPath，这个语法并不复杂。
